User flows, screens, and fields required for Web and Mobile App

Note: Use existing app design system. This document only describes the feature structure, flows, fields, and screens needed.

1. User Flows

1.1. Flow: Post an Opportunity

  1. User clicks "Post Opportunity" button (visible only to role_id = 4)
  2. System checks: Has user posted in last 24 hours?
  3. If yes â†’ Show error message: "You can only post 1 opportunity per 24 hours"
  4. If no â†’ Show create opportunity form
  5. User fills form and submits
  6. Opportunity created â†’ Redirect to opportunity detail page

1.2. Flow: Browse Opportunities

  1. User navigates to "Opportunities" page
  2. System shows list of open opportunities
  3. User can apply filters (industry, category, budget, location, skills)
  4. User can search by title/description
  5. User can sort (newest, deadline, proposals count)
  6. System shows skills-matched opportunities at top (if user has matching skills)
  7. User clicks on opportunity card â†’ View opportunity details

1.3. Flow: View Opportunity Details

  1. User clicks on opportunity
  2. System shows full opportunity details
  3. If user is owner:
    • Show "Edit" and "Delete" buttons
    • Show "View Proposals" button
  4. If user is not owner:
    • Check if already submitted proposal â†’ Show "View My Proposal"
    • Check if can submit (3 proposals in 24h limit) â†’ Show "Submit Proposal" or error
    • Show "Save" button

1.4. Flow: Submit Proposal

  1. User clicks "Submit Proposal" on opportunity detail page
  2. System checks:
    • Already submitted? â†’ Show error
    • Submitted 3 proposals in last 24 hours? â†’ Show error with countdown
    • Opportunity still open? â†’ Show error if closed
  3. If all checks pass â†’ Show proposal form
  4. User fills form and submits
  5. Proposal created â†’ Notification sent to owner â†’ Redirect to "My Applications"

1.5. Flow: Manage Proposals (Opportunity Owner)

  1. Owner navigates to "My Opportunities" â†’ Selects opportunity â†’ "View Proposals"
  2. System shows list of all proposals
  3. Owner can:
    • Click "Shortlist" â†’ Status changes to shortlisted â†’ Notification sent
    • Click "Accept" â†’ Status changes to accepted â†’ Opportunity status changes to awarded â†’ Notification sent
    • Click "Reject" â†’ Status changes to rejected â†’ Notification sent
    • Click "Contact" â†’ Opens chat with proposer
  4. Owner can filter proposals by status

1.6. Flow: View My Applications

  1. User navigates to "My Applications"
  2. System shows list of user's submitted proposals
  3. Each item shows: Opportunity title, Status, Submitted date
  4. User can filter by status
  5. User can click "Withdraw" if status is pending

1.7. Flow: Save/Unsave Opportunity

  1. User clicks save/bookmark icon on opportunity card or detail page
  2. System saves opportunity to user's saved list
  3. Icon changes to filled/bookmarked state
  4. User can view all saved opportunities in "Saved Opportunities" page

1.8. Flow: Skills Matching

  1. System extracts user's skills from profile/services/products
  2. System matches against opportunity required_skills
  3. Shows matched opportunities at top with "Recommended" badge
  4. Shows match percentage (e.g., "80% match")
  5. Highlights matched skills in skills tags

4. Screen Layouts & Structure

4.1. Browse Opportunities Page (Web)

Layout:

  • Header Section:
    • Search bar (full width)
    • Filter toggle button
  • Left Sidebar (280px):
    • Filter panel (collapsible sections):
      • Industry multi-select
      • Category checkboxes
      • Budget range slider
      • Location input
      • Skills tags input
      • Clear filters button
      • Apply filters button
  • Main Content Area:
    • Sort dropdown (Newest, Deadline, Proposals)
    • View toggle (Grid/List view)
    • Opportunity cards grid (3 columns desktop, 2 tablet, 1 mobile)
    • Pagination at bottom

4.2. Opportunity Card (Grid View)

Elements to Display:

  • Title (clickable, links to detail)
  • Category badge
  • Industry label
  • Budget range
  • Timeline
  • Skills tags (max 3 visible, "+X more" if more)
  • Stats: Proposals count, Views count
  • "Recommended" badge (if skills matched)
  • Save/Unsave icon button
  • "View Details" button

4.3. Opportunity Detail Page (Web)

Layout:

  • Breadcrumb: Home > Opportunities > [Title]
  • Main Content (Left 70%):
    • Title with status badge
    • Poster profile card (avatar, name, company, location)
    • Description section
    • Details grid:
      • Category, Industry, Budget, Timeline, Location
      • Required Skills (tags)
      • Preferred Experience
      • Deadline countdown
    • Action buttons: Submit Proposal, Save, Share
  • Right Sidebar (30%):
    • Quick stats card (views, proposals)
    • Related opportunities (3-5 cards)
    • Poster info card

4.4. My Opportunities Dashboard (Web)

Layout:

  • Stats Cards Row (Top):
    • Total Posted
    • Active Opportunities
    • Proposals Received
    • Total Views
  • Filter Tabs: All, Open, In Review, Awarded, Closed
  • Opportunity List:
    • Table or card view
    • Columns: Title, Status, Proposals, Views, Created Date, Actions
    • Actions: View, Edit, Close, View Proposals

4.5. Proposals Management Page (Web)

Layout:

  • Opportunity Header: Title, Status, Back button
  • Filter Tabs: All, Pending, Shortlisted, Accepted, Rejected
  • Proposal Cards:
    • Proposer profile (avatar, name, rating)
    • Proposal text (truncated, "Read more" link)
    • Proposed budget and timeline
    • Portfolio items (links/thumbnails)
    • Status badge
    • Submitted date
    • Action buttons: Shortlist, Accept, Reject, Contact

4.6. Browse Opportunities (Mobile)

Layout:

  • Header: Back button, "Opportunities" title, Filter icon
  • Search Bar: Full width, sticky on scroll
  • Filter Chips: Horizontal scrollable chips (Industry, Category, Budget)
  • Sort Button: Top bar or floating action
  • Opportunity Cards:
    • Full width cards
    • Stack vertically
    • Swipe actions: Save, Quick View
  • Bottom Navigation: Home, Opportunities, Saved, Profile

4.7. Opportunity Detail (Mobile)

Layout:

  • Header: Back button, Title, Share/Save icons
  • Content (Scrollable):
    • Status badge
    • Poster profile card
    • Description (expandable if long)
    • Details accordion sections
    • Skills tags
    • Deadline countdown
  • Sticky Bottom Bar: Submit Proposal button (full width)

3 proposals limit "You can only submit 3 proposals per 24 hours. Next submission available in [X hours]" When user tries to submit 4th proposal in 24 hours
Opportunity closed "This opportunity is no longer accepting proposals" When user tries to submit proposal on closed opportunity
Role restriction "Only members can post opportunities" When non-member tries to post
